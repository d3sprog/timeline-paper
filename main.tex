\begin{abstract}
Spreadsheets make it easy to express computations over two-dimensional data, but two dimensions
are not enough to express rich computations with time such as simulations, complex systems
models, analyses of historical data, or interactive systems. We present \Timeline, a system that
adds discrete time to spreadsheets.

The remarkable insight from our work is that a large number of advanced programming language concepts
can be directly applied in the context of spreadsheets with time. \Timeline draws from
\emph{dataflow languages} to express computations over time, \emph{coeffect systems} to store
only the necessary previous values, \emph{functional reactive programming} to support interactivity,
\emph{grammars of graphics} to support composable visualizations, and \emph{typed holes} for
inserting cell references in the formula editor.

In this paper, we provide an overview of the \Timeline design and discuss how it adapts the
aforementioned programming language innovations for the context of spreadsheets. We then formalize
the evaluation of spreadsheets with discrete time through a \emph{core calculus}, describe a
coeffect-based inference for computing the required number of past values and prove that the
optimization is sound. More broadly, this paper shows that established programming language
ideas can often be productively used outside of their original domain.

\end{abstract}

\section{Introduction}
Spreadsheets are sometimes referred to as ``the most widely used programming systems in the
world'' \cite{abraham-2009-prog}. Their uses range from personal productivity tools and decision
making support systems to scientific analytical instruments and financial reporting
systems \cite{grossman-2023-taxonomy}. However, spreadsheets are first-order, limited to tabular
data structures, and lack abstraction mechanisms.

Many extensions to the core spreadsheet model have been proposed, generalising the tabular
structure to more general forms \cite{burnett-2001-forms3,bazant-2018-nontabular}, adding
support for objects \cite{mccutchen-2016-oop}, or incorporating support for streaming data
to use spreadsheets as the back-end for web development \cite{chang-2014-web,chang-2015-streaming}.
Abstraction mechanism developed for spreadsheets include sheet-defined functions
\cite{jones-2003-functions,sestoft-2013-sheetdef} and gridlets \cite{joharizadeh-2020-gridlets},
while programming by example \cite{gulwani-2012-examples} simplifies data wrangling within
spreadsheets.

As the above extensions show, extending spreadsheets is a difficult balancing act. How can we make
spreadsheets more expressive, while preserving their liveness, directness and
usability \cite{hermans-2016-spreadsheets}? The aim of the \Timeline project is to extend
spreadsheets with a built-in notion of discrete time, making it possible to use spreadsheets
in a range of domains that are difficult to tackle with only two dimensions \cite{litt-2017-multidim}
such as physics simulations, agent-based modelling \cite{abar-2017-abm}, financial data analysis,
or simple interactive applications. As illustrated in \autoref{fig:teaser}, \Timeline adds an
implicit discrete time to the spreadsheet programming model and extends the formula language to
allow references to previous values and user interface events, but otherwise preserves the
standard spreadsheet model.

\subsubsection*{Adapting programming langauge research.}
Although \Timeline is interesting as an exploration of the expressive power of spreadsheets with
discrete time, this paper considers it primarily from the technical perspective of programming
language design and implementation. We show that a large number of concepts and techniques
developed in the context of programming language research are directly applicable to
spreadsheets.

The key addition to \Timeline is a built-in notion of discrete time. Formulas in cells can
refer to other cells (e.g.~\tml{G19}), but also values computed in earlier steps (e.g.~\tml{F19[-1]}).
The language design and runtime implementation of \Timeline draws inspiration from research on
\emph{synchronous dataflow languages} \cite{benveniste-2003-sync,halbwachs-1991-lustre}. To ensure
that simulations and interactive applications execute using bounded memory whenever possible,
we augment the runtime with a program analysis based on \emph{coeffects}
\cite{petricek-2013-coeffects,petricek-2014-coeffects}. The support for discrete times also makes
it possible to use the spreadsheet paradigm for building interactive applications. Here, we draw
on the \emph{functional reactive programming} paradigm~\cite{elliott-1997-fran,wan-2000-frp}.
Although not strictly related to time, \Timeline also adopts a \emph{composable data
visualization} model \cite{petricek-2021-compost,marasoiu-2019-cuscus,wilkinson-2012-gg}.
Finally, the \Timeline also uses a notion of \emph{typed holes} \cite{omar-2019-typedholes}
to determine when the user may want to insert a cell reference during formula editing.

\subsubsection*{Contributions} This paper presents the design and implementation of a
spreadsheet system with discrete time and captures its essence through a core calculus.

\begin{itemize}[itemsep=2mm,topsep=2mm]
\item We provide an overview of the \Timeline system design through a simple physics simulation
  example (\autoref{sec:walkthrough}) and then examine the expressive power of the programming model
  through a number of case studies covering agent-based modelling, simulations and finance
  (\autoref{sec:case-studies}).

\item We discuss how how the design and implementation of \Timeline adapts a range of advanced
  programming language concepts and techniques for the context of spreadsheet
  systems~(\autoref{sec:features}), including dataflow languages, coeffects,
  functional reactive programming, visualization grammars and typed holes.

\item We present a core calculus of the \Timeline execution model, which captures the way previous
  values are stored during the spreadsheet evaluation (\autoref{sec:calculus}). We formalize a
  space optimization based on coeffects implemented in \Timeline and prove its soundness
  (\autoref{sec:calculus-coeffects}).


\end{itemize}

\section{Overview}
\label{sec:walkthrough}

We overview \Timeline using a simple cannonball firing simulation (\autoref{fig:cannon}).
In each step, the simulation computes the cannonball position using the position in the
previous step and the current time step. We ignore air resistance, but this could be easily
added to a stepwise simulation model. The simulation stops when the cannonballs hit the ground.
